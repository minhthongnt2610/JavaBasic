Ôn tập OOP nâng cao

Bốn trụ cột của OOP

Java là một ngôn ngữ lập trình hướng đối tượng. Trong chương này, chúng ta đã tìm hiểu về 4 trụ cột chính của OOP.

  • Tính kế thừa
  • Tính trừu tượng (phương thức/lớp trừu tượng và interface)
  • Tính đa hình
  • Tính đóng gói

Chúng ta đã tìm hiểu chi tiết về Tính kế thừa trong chương trước.

Bây giờ, chúng ta sẽ ôn tập 3 khái niệm còn lại qua các ví dụ sau:

  • Triển khai hệ thống phanh xe máy
  • Nạp chồng phương thức thanh toán trong thương mại điện tử
  • Tính diện tích hình tròn sử dụng tính đóng gói

Bắt đầu nào.

Triển khai hệ thống phanh xe máy

Trong phần này, chúng ta sẽ xem một ví dụ về tính trừu tượng. Một ví dụ thực tế về tính trừu tượng là hệ thống phanh xe máy.

Cách thực hiện

Chúng ta biết rằng công dụng của phanh là để dừng xe máy. Tuy nhiên, nguyên lý hoạt động của các hệ thống phanh là khác nhau đối với các loại xe khác nhau.

Vì vậy, thay vì cung cấp một triển khai riêng biệt, chúng ta có thể làm cho nó trở nên trừu tượng (ẩn các chi tiết triển khai).

Giờ đây, tất cả các loại xe máy sẽ cung cấp triển khai hệ thống phanh riêng tương ứng. Tuy nhiên, công dụng chung của các hệ thống phanh là như nhau (để dừng xe máy).

Code

interface MotorBike {
    // hide the implementation of brake
    abstract void brake();
}
 
class SportsBike implements MotorBike {
    
    // provide an implementation of brake
    public void brake() {
        System.out.println("Stopping the Sports Bike.");
    }
}
 
class MountainBike implements MotorBike {
    
    // provide an implementation of brake
    public void brake() {
        System.out.println("Stopping the Mountain Bike.");
    }
}
 
class Main {
    public static void main(String[] args) {
        
        SportsBike s1 = new SportsBike();
        s1.brake();
        
        MountainBike m1 = new MountainBike();
        m1.brake();
    }
}

Đầu ra

Stopping the Sports Bike.
Stopping the Mountain Bike.

Trong ví dụ trên, chúng ta đã tạo interface MotorBike với phương thức trừu tượng brake().

Lớp SportsBike và MountainBike triển khai lớp MotorBike và cung cấp triển khai cho phương thức trừu tượng brake().

Nạp chồng phương thức thanh toán trong thương mại điện tử

Trong ví dụ này, chúng ta sẽ triển khai phương thức thanh toán cho một doanh nghiệp Thương mại điện tử.

Cách thực hiện

Chúng ta biết rằng mọi doanh nghiệp Thương mại điện tử đều có các phương thức thanh toán khác nhau như thanh toán bằng thẻ tín dụng, thanh toán qua PayPal, v.v.

Tất cả các phương thức thanh toán yêu cầu thông tin khác nhau. Ví dụ:

Để thanh toán qua thẻ tín dụng, ta cần có số thẻ, CVV và ngày hết hạn. Tương tự, để thanh toán qua Paypal, chúng ta chỉ cần id.

Hãy triển khai logic tương tự và nạp chồng phương thức thanh toán với số lượng tham số khác nhau.

Code

class Payment {
    
    // pay through credit card
    void makePayment(String cardNumber, String cvv, String expiryDate) {
        System.out.println("Payment Through Credit Card is Successful");
    }
    
    // pay through Paypal
    void makePayment(String id) {
        System.out.println("Payment Through PayPal is Successful");
    }
}
 
class Main {
    public static void main(String[] args) {
        
        Payment pay = new Payment();
        
        // make payment through credit card
        pay.makePayment("5415 3252 3523 6315", "131", "12/025");
        
        // make payment through PayPal
        pay.makePayment("97629639067");
    }
}

Lưu ý: Để cho đơn giản, chúng ta trực tiếp cung cấp giá trị cho các đối số của phương thức. Đây là một cách làm khuyên dùng để lấy đầu vào cho các giá trị này.

Tính diện tích hình tròn sử dụng Tính đóng gói

Trong ví dụ này, chúng ta sẽ tính diện tích hình tròn bằng cách sử dụng khái niệm đóng gói. Chúng ta sẽ sử dụng công thức sau:

Area of a circle = 3. 14 * radius * radius

Cách thực hiện

Từ công thức, chúng ta biết rằng cần phải có bán kính để tính diện tích. Ở đây, chúng ta sử dụng trường private radius, do đó nó không thể bị sửa đổi trực tiếp từ lớp bên ngoài.

class Area {
    private double radius;
}

Sau đó, ta sử dụng phương thức setter và getter để sửa đổi và truy cập giá trị của radius từ bên ngoài.

// setter method
public void setRadius(double radius) {
    
}

// getter method
public double getRadius()

}

Tuy nhiên, bên trong phương thức setter, chúng ta sẽ sử dụng điều kiện để giá trị radius không âm.

public void setRadius(double radius) {
 
    if (radius > 0) {
        this.radius = radius;
    }
    else {
        System.out.println("Radius is negative");
    }
}

Bây giờ, chương trình này đảm bảo rằng giá trị bán kính không âm.

Hãy hoàn thiện chương trình:

import java.util.Scanner;
 
class Area {
 
    private double radius;
 
    // setter method
    public void setRadius(double radius) {
        
        if (radius > 0) {
            this.radius = radius;
        }
        else {
            System.out.println("Radius is negative");
        }
    }
 
    // getter method
    public double getRadius() {
        return this.radius;
    }
}
 
class Main {
 
    public static void main(String[] args) {
 
        // object of Area class
        Area area = new Area();
        
        // get radius input
        System.out.println("Enter the value of radius:");
        Scanner input = new Scanner(System.in);
        double radius = input.nextDouble();
        
        // set value of private radius using setter
        area.setRadius(radius);
 
        // access the value of radius and compute the area
        double circleArea = 3.14 * area.getRadius() * area.getRadius();
        System.out.println("Area of Circle: " + circleArea);
    }
}

Kết quả mẫu 1

Enter the value of radius:
8
Area of Circle: 200.96

Kết quả mẫu 2

Enter the value of radius:
-8
Radius is negative
Area of Circle: 0.0

Chương trình chỉ gán giá trị cho radius, nếu nó không âm. Nếu không, giá trị mặc định của double 0.0 sẽ được gán cho bán kính. Bằng cách này, chúng ta có thể kiểm soát chương trình bằng cách không cho phép người dùng nhập các giá trị âm.
